# NtE 素材管理ツール

Neverness to Everness（NtE）の **最大レベル / スキルレベル / 弧盤** に必要な
素材を、現在の進捗から「残り素材数」と「全キャラ合計（全素材数）」として
集計するデスクトップアプリです。

- 言語: Python 3.10+（標準ライブラリのみ・追加インストール不要）
- GUI: Tkinter
- データ保存: JSON（`~/.nte_material/data.json` に自動保存・次回復元）

## 起動方法

```sh
python main.py
```

## 仕組み

必要数は **全キャラ共通** で、キャラごとに変わるのは「どの素材を使うか（種類）」
だけ、という NtE の仕様に基づきます。キャラ登録時に素材の種類を選ぶと、
共通の必要数テーブル（`src/nte_material/gamedata.py`）から具体名へ解決して
集計します。

### 3カテゴリと素材

| カテゴリ | 進捗入力 | 使う素材 |
|---|---|---|
| 最大レベル | 現在Lv（20〜80のドロップダウン） | 異象狩り素材 ＋ 育成素材(本体ライン) |
| スキルレベル | 4スキル各Lv1〜10 ＋ サポート追加パッシブ2種 | カード ＋ 育成素材(本体ライン) ＋ 異象巡礼 |
| 弧盤 | 現在Lv（20〜80） | 専用素材 ＋ 育成素材(弧盤ライン・本体と独立) |

- 育成素材ラインは4種（呟き／姿／数字記号／ファントム）。本体は最大Lv＋スキルで
  共通、**弧盤だけは別ライン**を選べます（モチーフ以外のアークを装備し得るため）。
- ビートルコイン・Fons・Dreamless Seeds は集計対象外です。

### 集計ルール

- 最大レベル・弧盤: 「現在Lv以上の突破段階」の素材が残り。80＝完了で残りゼロ。
- スキル: 「現在Lvより上の段階(Lv2〜10)」の素材が残り。
- サポート追加パッシブ: 未開放（チェックなし）なら残りに加算。
- 各カテゴリ（最大レベル / スキル / 弧盤）に「集計に含める」チェックがあり、
  外したカテゴリは残り集計から除外される（既定はすべてオン）。

## 画面（2タブ）

### 育成計画タブ

1. 左の「追加」でキャラを登録。
2. 中央「素材の種類」で、このキャラが使う育成素材ライン・異象狩り・カード・
   異象巡礼・弧盤の専用素材／育成素材ラインを選択。
3. 中央「進捗」で最大レベル・弧盤レベル・各スキルLv・サポートパッシブを入力。
4. 右側に「このキャラの残り素材」と「全キャラ合計（残り / 所持 / 不足）」を表示。

### 所持リソースタブ

各素材の所持数を管理します。素材を選んでから:

- **直接設定**: 数値を入れて「設定」で所持数を上書き。
- **調整量 ＋／－**: 入力した数だけ増減（一定数の調整）。
- **新規素材＋追加**: 一覧に無い素材（保留中の Dreamless Seeds 等）を追加。
- **選択を削除**: 所持リストから素材を削除。

所持数を入れると、育成計画タブの「不足（＝残り−所持）」に反映されます。
入力は変更のたび JSON へ自動保存されます。

## 素材アイコン

各素材行にアイコンを表示します。

1. `assets/icons/<素材名>.png` があればそれを表示
2. 無ければレア度色（緑/青/紫）のプレースホルダを自動生成

実画像は差し替え可能な別資産として扱います（コードは画像に非依存）。
`assets/icon_sources.json` に全素材の Wiki 画像URL（wikiwiki.jp の webp）を
記入済みなので、`python tools/fetch_icons.py` でダウンロード＋PNG変換できます
（webp→PNG変換に **Pillow** が必要: `pip install pillow`）。詳細・権利の注意は
[assets/README.md](assets/README.md) を参照。

> `assets/icons/*.png` は `.gitignore` 済み。リポジトリには画像を同梱せず、
> 再配布時は `assets/icons` を空にすればプレースホルダ表示に戻ります。

## ソース構成

```
src/nte_material/
  gamedata.py   必要数テーブルと素材名（ここを直せばデータ調整できる）
  models.py     キャラ状態 + 残り素材 + 所持リソースの集計ロジック
  storage.py    JSON 保存/読込
  icons.py      アイコン読み込み（assets優先・無ければ自動生成）
  app.py        Tkinter GUI（育成計画 / 所持リソース の2タブ）
tools/fetch_icons.py  アイコン一括ダウンロード
assets/icon_sources.json  素材名→画像URL の対応表
tests/  集計・アイコン補助の検証（判明値との一致テスト含む）
```

## テスト

```sh
python tests/test_models.py     # 追加ライブラリ不要
# または
python -m pytest
```

## データ出典

必要数は以下を相互検証（カオス／ラクリモサ／ナナリで一致確認）:
oslink・lootbar・GameWith。スキルの段階別内訳は GameWith（ナナリ）。
`gamedata.py` の数値はゲーム更新で変わり得るため、必要に応じて調整してください。
